# v0.1.0-beta.1 — public beta

> **This is a beta. Not published to npm. Install from source.**
> The two-tool surface, the QuickJS sync + Promise-callback host
> bridge, the typed query / mutation dispatcher, the bundled SDL
> fallback (pinned to a tagged `unraid/api` release), and the
> single- and multi-user credential modes are all implemented and
> green in CI. Live verification so far covers a single Unraid 7.2
> box (the maintainer's homelab) through the raw stdio harness —
> every agent / IDE client (Cursor, Claude Code, Claude Desktop,
> VS Code + Copilot, Codex CLI, Continue, Cline, opencode, MCP
> Inspector, …), the Streamable HTTP transport in multi-tenant
> mode, the Cloudflare Workers entry, and any non-VM mutation are
> wired but **NOT verified by us**. See [`README.md`](../../README.md)
> and [`CONTRIBUTING.md`](../../CONTRIBUTING.md) for how to participate.

A code-mode MCP server for the **Unraid 7.2+ GraphQL API**. Two MCP
tools — `search` and `execute` — let an LLM agent introspect and
call any Unraid GraphQL field by writing JavaScript that runs inside
a sandboxed QuickJS WASM context. Cloudflare's Code-Mode pattern
applied to Unraid's ~57 queries × ~45 mutations × 240+ types — your
LLM context stays small no matter how big the schema is.

Single-user (env vars) and multi-user (per-request `X-Unraid-*`
HTTP headers) modes. Stdio + Streamable HTTP transports. Cloudflare
Workers entry (scaffolded). Strict TLS by default with per-tenant
overrides.

## What you can do

```js
// Typed query — typed dispatch synthesises the GraphQL document
const info = await unraid.local.query.info({
  fields: ['os { distro release kernel uptime }', 'cpu { brand cores threads }'].join(' '),
});

// Sequential awaits inside one execute() — fully supported
const arr = await unraid.local.query.array({ fields: 'state' });
const shares = await unraid.local.query.shares({ fields: 'name free used size' });

// Or batch them in parallel
const [vms, docker, online] = await Promise.all([
  unraid.local.query.vms({ fields: 'domain { uuid name state }' }),
  unraid.local.query.docker({ fields: 'containers { id names state }' }),
  unraid.local.graphql({ query: 'query { online }' }),
]);

// Mutation — same dispatch shape (verified live with vmStart / vmStop)
const result = await unraid.local.mutation.vmStart({ args: { id: '<vm-uuid>' } });

// Raw GraphQL escape hatch
const data = await unraid.local.graphql({
  query: 'query { array { state capacity { kilobytes { free total } } } }',
});

// Raw HTTP escape hatch for the rare non-GraphQL endpoints
const probe = await unraid.local.request({ method: 'GET', path: '/health' });
```

The full surface is documented in [`SKILL.md`](../../SKILL.md) (the
operating manual the LLM reads) and [`docs/usage.md`](../usage.md)
(for humans).

## What's wired but NOT yet verified

We want bug reports and verification reports for these in particular:

- **Any agent / IDE client.** All live verification so far has been
  through `scripts/mcp-call.mjs` driving the stdio transport
  directly. Cursor (chat panel), Claude Code, Claude Desktop, VS Code
  + Copilot, Codex CLI, Continue, Cline, opencode, Aider, Zed, MCP
  Inspector (CLI + UI) — all wired but **NOT verified by us**.
  End-to-end LLM-mediated invocation is the most useful thing
  testers can report on. Use the [verification report template](../../.github/ISSUE_TEMPLATE/verification_report.yml).
- **Streamable HTTP transport in multi-tenant mode.** The transport
  is wired and unit-tested; a real multi-tenant deployment behind a
  reverse proxy with header-based credentials is not.
- **Cloudflare Workers entry.** `cf-worker/` is scaffolded against
  `@cloudflare/codemode` but the Web `Request`/`Response` ↔ MCP SDK
  Node-stream adapter is not implemented, and the Worker is not
  deployed anywhere. Tracked in [`cf-worker/README.md`](../../cf-worker/README.md).
- **Mutations beyond VM start/stop.** Docker container `start` / `stop`,
  parity check `start` / `cancel`, share / disk operations, user /
  API-key management, and the full mutation surface are wired through
  the typed dispatcher but **not live-verified**. Probing them
  blindly against live hardware is unsafe; we want testers with
  redundant homelabs.
- **Unraid Connect cloud surface.** `unraid.connect.*` is reserved in
  `TenantContext` and not yet implemented.
- **Real Unraid boxes other than the maintainer's homelab.** A
  single Unraid 7.2 box is not generalisable evidence — different
  array configs, plugin sets, network topologies, and Unraid
  versions will all surface different edge cases.
- **Long-running soak / stability under sustained load.**

See the README's [verification status](../../README.md#verification-status)
table for the full matrix.

## Highlights

- **Two tools.** `search` is read-only (no network); `execute` runs
  sandboxed JS that calls real Unraid GraphQL.
- **QuickJS sync + Promise-callback host bridge** (instead of asyncify).
  Synchronous host functions accept sandbox-side `resolve` / `reject`
  handles, kick off Node async work, and invoke the sandbox callbacks
  on completion. The host's drain loop alternates `executePendingJobs()`
  with `setImmediate` and uses a wall-clock deadline. The earlier
  asyncify-based implementation hit known upstream
  `quickjs-emscripten` heap-corruption bugs (#258, #261, #235) on
  multi-await scripts; this is the canonical workaround. **Sequential
  `await` and `Promise.all` both work cleanly** — no per-call
  iteration cap, no premature timeout, no heap corruption.
- **Typed convenience calls.** `unraid.local.query.<fieldName>({ args, fields })`
  and `unraid.local.mutation.<fieldName>({ args, fields })` synthesise
  the GraphQL document for you using introspected arg types.
- **Raw escape hatches.** `unraid.local.graphql({ query, variables })`
  posts any document; `unraid.local.request({ method, path, body })`
  covers the rare non-GraphQL endpoints.
- **Single- and multi-user modes.** Single-user via env vars (stdio),
  multi-user via `X-Unraid-Api-Key` HTTP headers with
  `AsyncLocalStorage`-scoped credentials so concurrent requests
  can't bleed.
- **Per-tenant TLS.** Custom CA bundles or explicit `insecure` opt-in
  survive into a per-request `undici` dispatcher. The Cloudflare
  Worker variant rejects insecure mode entirely.
- **Bundled SDL fallback.** Server boots without an Unraid box; the
  bundled SDL is **pinned to a tagged `unraid/api` release**
  (currently `v4.33.0`) — no `main`-drift. `scripts/update-spec.ts`
  refreshes from the pinned tag and stamps a header with source URL,
  pinned tag, and fetch timestamp.
- **Sandbox limits.** 64 MiB memory, 30 s wall-clock timeout, 50
  API calls per `execute()` (configurable via
  `UNRAID_MAX_CALLS_PER_EXECUTE`).

## Install

```bash
git clone https://github.com/jmpijll/unraid-code-mode-mcp.git
cd unraid-code-mode-mcp
npm install --legacy-peer-deps
cp .env.example .env  # set UNRAID_BASE_URL and UNRAID_API_KEY
npm run build
npm start
```

For the multi-tenant HTTP transport: `MCP_TRANSPORT=http npm start`
and pass `X-Unraid-Api-Key` / `X-Unraid-Base-Url` (and optionally
`X-Unraid-Insecure` / `X-Unraid-Ca-Cert`) headers per request. See
[`docs/multi-tenant.md`](../multi-tenant.md).

To wire it into Cursor:

```json
{
  "mcpServers": {
    "unraid": {
      "command": "node",
      "args": ["/absolute/path/to/unraid-code-mode-mcp/dist/index.js"]
    }
  }
}
```

(See [`docs/deployment.md`](../deployment.md) for Docker / systemd /
Cloudflare Workers.)
