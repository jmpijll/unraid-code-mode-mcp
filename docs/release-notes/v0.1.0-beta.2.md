### `UNRAID_EXECUTE_TIMEOUT_MS` env var (closes a roadmap item)

Override the sandbox wall-clock deadline for `execute` invocations. Range 1 s – 10 min, default 30 s.

```bash
# Slow-booting VM; give the sandbox 5 minutes
UNRAID_EXECUTE_TIMEOUT_MS=300000 node dist/index.js
```

Plumbed through `src/config.ts → src/index.ts → src/sandbox/limits.ts → src/sandbox/execute-executor.ts` with new `src/__tests__/config.test.ts` covering parsing + range bounds. To use it from the Cursor / opencode configs, add it to the `env` / `environment` block in `.cursor/mcp.json` / `opencode.json`.

### CSRF-aware error decoration

When the upstream Unraid box returns `extensions.code: UNAUTHENTICATED` with an "Invalid CSRF token" payload (HTTP 401 wrapped inside a structured GraphQL error body), the MCP server now decorates the error returned to the agent with a clear remediation hint:

```text
GraphQL error: Invalid CSRF token. Hint: this Unraid box rejected an
`x-api-key`-authenticated request with a CSRF / UNAUTHENTICATED error.
Most Unraid 7.2 boxes accept the API key without a CSRF token; if yours
does not, re-create the API key (Settings → Management Access → API Keys),
confirm `x-api-key` works with `curl -H "x-api-key: <key>" -d
'{"query":"{ online }"}' <BASE_URL>/graphql`, and check the box's
`unraid-api` logs for the CSRF middleware decision.
```

We've seen this policy flip mid-session on a single Unraid 7.2 box — see the [opencode verification transcript](../../out/verification/opencode-deepseek-mcp-call.txt) for the live capture and [`docs/security.md`](../security.md#unraid-72-csrf-behaviour) for the operator runbook (when to re-mint, where to look in `unraid-api logs`, etc).

### Server version reads from `package.json` at runtime

The MCP `serverInfo.version` in the `initialize` handshake was hardcoded to `0.1.0`, which drifted as soon as we tagged `0.1.0-beta.1`. Now it reads from `package.json` (`src/index.ts → readPackageVersion()`), so the wire-level version always matches the package version. The cf-worker entry's hand-stamped version is documented as a known limitation (Workers can't read `package.json` at runtime) and is bumped alongside the Node entry per release.

### Smaller fixes

- **`scripts/mcp-call.mjs`** does a clean shutdown — half-closes stdin only after a 50 ms grace window for the server to flush queued writes, and explicitly swallows EPIPE on stdin so an early server shutdown doesn't crash the script. Resolves the noisy EPIPE traces we saw during the 2026-05-09 confirmation probes.
- **`CLAUDE.md`** at the repo root, redirecting Claude Code to the canonical `AGENTS.md`. No content duplication; `AGENTS.md` remains authoritative.
- **`docs/security.md`** picked up the new env var and the new "Unraid 7.2 CSRF behaviour" section.

## What's still open

See the [README's Roadmap section](../../README.md#roadmap). The high-leverage open items are:

1. More LLM clients verified (Claude Code CLI, Claude Desktop, MCP Inspector UI, VS Code + Copilot)
2. Auto-bump bundled SDL pin via Dependabot-style PRs (foundational)
3. Streamable HTTP multi-tenant deployment behind a real reverse proxy
4. Mutation matrix beyond VM start/stop (Docker, parity, share/disk)
5. `unraid.connect.*` namespace
6. Cloudflare Workers transport adapter (blocked on upstream MCP SDK)
7. NPM publish (reserved for `1.0.0`)

## How to upgrade

```bash
git pull
npm install --legacy-peer-deps
npm run build
```

If you set `UNRAID_EXECUTE_TIMEOUT_MS` in your `.env` / agent config, it'll just work. No env vars were renamed or removed.
