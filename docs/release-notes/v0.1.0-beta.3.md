# v0.1.0-beta.3 — "Maintenance + automation"

> **This is a beta. Not published to npm. Install from source.**
> Maintenance + automation release. No behaviour changes — all-tooling.

## What's new

### Auto-bump the bundled SDL pin (closes a roadmap item)

`.github/workflows/update-spec.yml` runs every Monday at 06:37 UTC (and on `workflow_dispatch`). When a newer [`unraid/api`](https://github.com/unraid/api/releases) release exists than the current `PINNED_UNRAID_TAG` in [`scripts/update-spec.ts`](../../scripts/update-spec.ts), the workflow:

1. Bumps the pin in the script.
2. Regenerates `src/spec/local-fallback.graphql` against the new tag.
3. Runs `lint`, `typecheck`, `test` against the new SDL (non-blocking — we want the PR to open even if tests fail so a human can investigate).
4. Opens a PR labelled `dependencies` + `spec`, assigned to the maintainer, with a release-notes link and the full upstream diff URL.

The PR has a reviewer checklist (eyeball the schema diff, regenerate any docs that reference specific operation names, run the local live-verification matrix before tagging the next release).

If you fork this repo, the workflow runs on your fork too — same upstream feed, same automation. No setup required beyond the existing `peter-evans/create-pull-request@v7` action.

### Local MCP Inspector smoke (`npm run smoke:inspector`)

```bash
npm run build
npm run smoke:inspector
```

Mirrors the CI smoke — boots `dist/index.js`, asks for `tools/list`, asserts both `search` and `execute` are exposed. No Unraid box required (the bundled SDL fallback covers the schema). Useful as a fast pre-commit confidence check, especially after dependency updates or sandbox changes. CI now reuses the same script (single source of truth).

The script lives at [`scripts/smoke-inspector.sh`](../../scripts/smoke-inspector.sh); the CI step that calls it is the only line that changed in `.github/workflows/ci.yml`.

### "Verifying your install" section in the README

The new [README "Verifying your install" section](../../README.md#verifying-your-install) lists the four no-Unraid-needed local checks for newcomers:

```bash
npm run build
npm test                  # 65 unit + integration tests
npm run test:sandbox      # QuickJS host-bridge stress
npm run smoke:inspector   # MCP Inspector CLI smoke against built dist
```

If `smoke:inspector` prints `OK: both 'search' and 'execute' tools exposed by dist/index.js`, your build is wire-compatible with any MCP client.

### Repo settings: missing labels added to `post-flip-repo-settings.sh`

The `spec`, `dependencies`, `npm`, and `github-actions` labels (referenced by the new SDL-bumper workflow + Dependabot) are now created by `scripts/post-flip-repo-settings.sh` so they survive a fork/repo-recreate.
